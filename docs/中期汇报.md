## 四、跑分数据（基础有效性验证）

### 图像（参考为 `ref.png`）
| 失真类型 | PSNR ↑ | SSIM ↑ | NIQE-Lite ↓ |
|---|---:|---:|---:|
| 原图 | 100.000 dB | 1.0000 | 3.656 |
| 高斯模糊 (15×15, σ=4) | 25.534 dB | 0.6870 | **6.612** |
| 高斯噪声 (σ=25/255) | 24.020 dB | **0.4874** | 6.358 |
| JPEG q=12 | 29.382 dB | 0.7426 | 3.376 |

### 视频（参考为 `ref.mp4`，前 30 帧）
| 失真类型 | PSNR ↑ | SSIM ↑ | NIQE-Lite ↓ |
|---|---:|---:|---:|
| 原视频 | 100.000 dB | 1.0000 | 3.557 |
| 视频 高斯模糊 | 27.194 dB | 0.7329 | 4.575 |
| 视频 高斯噪声 | 27.564 dB | 0.6448 | 4.562 |

**观察结论**
1. PSNR、SSIM 在所有失真上严格低于无失真情况 → **基本单调性正确**。
2. NIQE-Lite 在模糊/噪声上得分明显升高（3.66 → 6.61 / 6.36），证明无参考通路可用；JPEG q=12 出现"高频被抹平 → NIQE 反而下降"的已知现象，将在后期接入官方 NIQE / VSFA 时解决（见 §6）。
3. 单图打分耗时 < 20 ms，30 帧视频 SSIM 耗时 ≈ 0.2 s；GUI 上交互流畅。

## 五、关键技术点

### 5.1 多线程不卡 UI
- `vqa.gui.worker.ScoringWorker` 继承 `QObject`，通过 `moveToThread(QThread)` 跑在后台线程；
- 进度、日志、完成、失败均使用 `pyqtSignal` 跨线程回到 UI；
- 界面层在任务进行时禁用"开始评估"按钮 + 进度条进入 busy 模式，**消除了视频处理时主线程冻结**这一开题中明确点名的难题；
- 已在离屏环境通过 `gui_smoke_test.py` 自检：MainWindow → 工作线程 → 完成信号链路无异常，SSIM 结果与 CLI 完全一致 (0.6870)。

### 5.2 SSIM 实现
- 直接对应 Wang 等 2004 的卷积形式：用 11×11 高斯窗滑动得到 μ、σ²、σ_xy；
- 通过 `cv2.filter2D` 而非自己写循环，保证速度 + 数值稳定。

### 5.3 NIQE-Lite（自实现轻量无参考）
故意不依赖任何预训练模型与 PCA 参数文件，特征构造：
1. **MSCN 系数**的全局方差与峰度——失真越严重越偏离自然分布；
2. MSCN 水平/垂直**乘积**的峰度——捕捉模糊与噪声；
3. **拉普拉斯响应方差**——模糊敏感；
4. **Immerkaer 噪声估计**——噪声敏感。

得分越大代表越偏离自然图像。优势：纯 NumPy/OpenCV、跨平台无障碍；劣势：与官方 NIQE 量纲不一致、对压缩失真不敏感——这正是后期切换到 VSFA / 官方 NIQE 要解决的。

### VMAF 调研要点（已完成阅读）
- VMAF = VIF（视觉信息保真度，4 个尺度）+ DLM（细节损失指标）+ TI（时域信息）→ SVM 回归；
- 默认模型：`vmaf_v0.6.1` 训练于 1080p；视频 SDR/HDR 各有专门模型；
- **集成路径推荐**：`pip install vmaf-python` 方式过老，更稳定的是把 `ffmpeg -lavfi libvmaf` 包成子进程，把 stdout 实时解析为逐帧得分；
- 因 GUI 已采用 QThread + 信号通路，VMAF 接入只需新增一个 `vqa/algos/vmaf.py` 实现 `score_video(ref, dis)` 即可，不破坏既有 API。
