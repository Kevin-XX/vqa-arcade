## 二、口播稿（直接照念）

### ① 开场 · 设计目标（15 秒）

> "GUI 这一块，我的设计出发点很明确——**让算法同学不用再写脚本、不用再翻 log，拖两个文件进来就能看到打分**。所以整个界面我就抓三个关键词：**对照直观、参数可调、结果可读**。"

### ② 四区布局（35 秒）—— 边讲边用激光笔指

> "界面我切成了四个功能区：
>
> - **左上算法区**：一键切换 SSIM、PSNR、NIQE-Lite，FR / NR 两种模式自动识别；
> - **中间双预览区**：左参考、右失真，**同步播放**，肉眼差异一目了然；
> - **右上参数区**：stride、max-frames 这些采样参数我都做成滑杆，调完即时生效；
> - **下方结果区**：左边是进度条和帧计数，右边是均值、方差、min/max 四个统计卡，再加一条帧级分数曲线。"

### ③ 交互亮点（25 秒）—— 这里切到 Slide 8 的 GIF/截图

> "交互上有三个我自己比较满意的小细节：
>
> 1. **拖拽即载入** —— MP4、MOV、PNG 都吃，扔进去自动判别该走 FR 还是 NR；
> 2. **多线程不卡 UI** —— 算法跑在 QThread 里，通过 pyqtSignal 把每帧分数实时推回主线程；
> 3. **一键导出** —— CSV / JSON / 截图三种格式，直接贴报告。"

### ④ 技术实现（20 秒）

> "技术栈是 **PyQt5 + QThread + pyqtSignal**。三个核心 widget —— preview、param_panel、score_panel —— 完全解耦，加新算法只要往算法注册表里挂一行。
>
> 还有个工程上的彩蛋：**整套 GUI 支持 offscreen 模式跑**，我今天给大家看的演示 GIF 就是用 `QT_QPA_PLATFORM=offscreen` 在没有显示器的环境下自动截屏出来的，将来接 CI 也很方便。"

### ⑤ 收尾金句（10 秒）

> "**一句话总结：把"写脚本 + 看 log"的研发流程，压缩成了"拖文件 + 看图表"。**"

## 三、评委追问预案（脑里备好）

| 评委可能问 | 你可以这样答 |
|---|---|
| 为什么不用 Web 前端做？ | 算法同学本地跑视频，离线、低延迟、零部署成本，PyQt5 比 Electron 启动快、内存占用低一半。 |
| 多线程怎么避免 UI 卡死？ | 算法在 QThread 里跑，通过 pyqtSignal 单向推分数；主线程只做渲染，从不直接调算法函数。 |
| 拖进来一张图和一段视频怎么处理？ | 入口统一抽象成 `frame_iter`，图片当成 1 帧的视频，算法层完全感知不到差异。 |
| 加新算法成本多大？ | 实现 `BaseMetric` 接口（compute / mode 两个方法），到注册表加一行字典映射，GUI 算法下拉框自动出现。 |
| 帧级分数曲线怎么画的？ | pyqtgraph 的 PlotWidget，每收到一个 pyqtSignal 就 append 一个点，零延迟刷新。 |
